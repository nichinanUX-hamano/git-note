## 記事の概要：MCPとAPIの違いをわかりやすく解説

---

### APIとは？

APIは「Application Programming Interface」の略で、ソフトウェア同士が会話するための「共通ルール」のことです。レストランのメニューに例えると、客（アプリA）がメニュー（API）を見て注文し、キッチン（アプリB）が料理を届ける仕組みです。内部の処理を知らなくても、決められた形式でリクエストを送れば結果が返ってくる、1対1の通信が基本です。

---

### MCPとは？

MCPは「Model Context Protocol」の略で、Anthropicが2024年末に発表した、AIモデルが外部のツールやデータソースと接続するための標準プロトコルです。USBポートに例えるとわかりやすく、バラバラだった接続端子が USB という統一規格で統一されたように、MCP はあらゆるツールを AI に接続する「統一の差込口」として機能します。

---

### MCPとAPIの5つの違い

1. **目的** — APIはあらゆるソフトウェア間の汎用的な通信の仕組み。MCPはAIモデルが外部ツールに接続するための専用規格。
    
2. **通信構造** — APIは「クライアント → サーバー」の1対1通信。MCPは「AIモデル ↔ MCPクライアント ↔ MCPサーバー群」という3層構造で、複数のツールを統一管理できる。
    
3. **誰のため？** — APIは開発者がコードを書いて実装するもの。MCPはAIモデル自身が自律的に使いこなすために設計されている。
    
4. **ツールの発見性** — APIでは開発者がドキュメントを読んで理解する必要がある。MCPではAIが接続先に「何ができますか？」と問い合わせると、ツール一覧と使い方が自動で返ってくる。
    
5. **新規サービスの追加** — APIは連携先ごとに個別実装が必要。MCPは同じパターンで接続するだけでよい。
    

---

### コードで見る違い

従来のAPIでは Slack・Google Calendar・GitHub それぞれで認証方式・URL・パラメータ形式がすべて異なり、バラバラの実装が必要です。MCPでは `connect()` で接続して `call_tool()` で実行するという同じパターンを繰り返すだけで、どのサービスとも連携できます。

---

### MCPの革命的な強み

MCPでは接続先サーバーに「何ができますか？」と問い合わせると、ツール名・説明・必要パラメータがすべて標準フォーマットで自動的に返ってきます。AIがツールの使い方を自分で読んで判断できる——これが「AIネイティブなプロトコル」と呼ばれる理由です。

---

### よくある誤解：MCPはAPIの代替ではない

MCPはAPIを置き換えるものではありません。MCPサーバーの内部では、実際には Slack API や Google Calendar API など従来のAPIが呼ばれており、MCPはAPIの上位レイヤーとして機能しています。バラバラだった各種APIを「AIが統一的に扱える形」にラッピングしている規格というのが正確な理解です。

---

### まとめ

|比較|API|MCP|
|---|---|---|
|たとえ|レストランのメニュー|USBポート|
|対象|あらゆるソフトウェア|AIモデル専用|
|通信|1対1|1対多を統一管理|
|ツール発見|開発者が手動で確認|AIが自動認識|
|相互関係|MCPの内部で使われる|APIの上位レイヤー|
