## 1. ナレッジグラフの定義
ナレッジグラフ（KG）とは、知識を単なるデータの集まりとしてではなく、**「もの（実体）」と「つながり」**によって表現する手法です。

* **ノード（Node）**: 実体（もの、概念、人など）を表します。
* **エライ（Edge）**: 実体どうしの関係性を表します。

例えば、「鈴木さんは株式会社ABCに勤めており、同社は東京に拠点を置いている」という情報を、複数の要素が線でつながった図として表現します。

## 2. 本質的な考え方：「文字列ではなく、実体を」
Googleが提唱した **"Things, not strings."（文字列ではなく、実体を）** という言葉が、KGの本質を表しています。

従来の検索エンジンなどは「東京」という「文字列」を扱っていましたが、KGは「東京」という**「実体そのもの」**（緯度・経度、人口、歴史的事実などの属性を持ったもの）として扱います。これにより、AIは単なる言葉のパターンではなく、意味のある知識として情報を理解できるようになります。

## 3. AI（特にLLM）にもたらす3つの価値
LLM（大規模言語モデル）の弱点を補う構造として、KGは以下の価値を提供します。

1. **推論の透明性**: 「なぜその回答になったのか」という根拠（エッジ）を辿ることができ、回答のプロセスをトレース可能です。
2. **更新性**: 学習済みのLLMとは異なり、データベースと同様に情報を追加・修正・削除できるため、常に最新の知識を反映できます。
3. **信頼性の向上**: 金額や権限などの「間違えてはいけない情報」をKGから直接取得することで、LLMのハルシネーション（もっともらしい嘘）を抑制し、役割分担（LLMは文章生成、KGは事実の提供）を実現できます。

## 4. 表現形式と構造
* **トリプル（Triple）**: 知識を「主語（Subject） - 述語（Predicate） - 目的語（Object）」の3要素で表現する最もシンプルな形式です（例：`<鈴木一郎> <所属する> <株式会社テックサービス>`）。これはRDFという仕様に基づいています。
* **オントロジー（Ontology）**: KGに「型（タイプ）」を導入することです。ノードに「従業員」や「会社」といったラベルを付け、エッジに「所属する」といったタイプを定義することで、情報の曖昧さを排除し、正確なクエリ（検索）を可能にします。

## 5. 他の技術との比較

| 比較対象 | 違いと特徴 |
| :--- | :--- |
| **RDB（リレーショナルDB）** | RDBは複雑な関係を追う際に大量の「JOIN（結合）」が必要になり、パフォーマンス低下やスキーマ変更の困難（JOINの壁）が生じますが、KGはエッジを辿るだけで自然に解決できます。 |
| **ベクトルDB** | ベクトルDBは「意味が似ている文書」を探す（意味検索）のが得意ですが、KGは「AとBの具体的な関係パス」を辿る（構造推論）のが得意です。両者は補完関係にあります。 |

## 6. 課題とトレードオフ
KGの導入には以下の課題が存在します。

* **構築コスト**: 正確なスキーマ設計や、既存データの変換（ETL）に高い投資が必要です。
* **設計の難しさ**: どのような関係性を定義するかという設計判断が重要であり、変更にもコストがかかります。
* **非構造化データへの弱さ**: 会議録やメールなどのテキストデータを直接格納・構造化するのは困難です。
