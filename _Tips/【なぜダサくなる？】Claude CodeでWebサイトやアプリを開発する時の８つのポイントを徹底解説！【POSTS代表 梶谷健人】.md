# 【要約】Claude CodeでWeb制作・アプリ開発を成功させる8つの鉄則

POSTS代表の梶谷健人氏による、Claude Codeを使って「素人感」のない高品質なプロダクトを効率的に開発するためのポイントです。

---

### 1. 「プランモード」での徹底した壁打ち

- **内容**: いきなりコーディングを始めず、まずはClaude Codeの「プランモード」でAIと対話します。
    
- **メリット**: 実装前に仕様書や計画を練ることで、自分の理想とAIのアウトプットのズレを最小化できます。

### 2. 「plan.md」による記憶の管理

- **内容**: `plan.md` というマークダウンファイルを作成し、仕様、技術選定、進捗を記録します。
    
- **重要性**: AIはセッションごとに記憶がリセット（記憶喪失）される特性があるため、このファイルを参照させることで文脈を維持します。

### 3. 開発を複数フェーズに分割する

- **内容**: 100%の完成形を一気に作らせようとせず、機能を20%ずつなどの部品に分けて段階的に実装します。
    
- **メリット**: 一箇所を直すと他が壊れる「モグラ叩き」状態を防ぎ、バグの発生を抑えられます。

### 4. デザイン特化AI「v0」との併用

- **内容**: 最初のデザイン（発射台）は、Webデザインに強いAI「v0」に作成させ、そのコードをClaude Codeに渡します。
    
- **メリット**: コーディングに慣れていない人でも、最初から一定以上のデザインレベルを担保できます。

### 5. コンテキストを伝えるための音声入力

- **内容**: 指示出しに音声入力を活用します。
    
- **メリット**: タイピングよりも情報量が多くなり、自分の頭の中にある細かいニュアンスや背景情報をAIに100%伝えやすくなります。

### 6. AIと相性の良い技術（shadcn/ui等）の活用

- **内容**: `shadcn/ui` のような、あらかじめ洗練されたUIコンポーネントライブラリの使用を指定します。
    
- **メリット**: ゼロからUIを作らせる無駄を省き、プロ仕様のパーツを組み合わせるだけで見栄えが良くなります。

### 7. ドキュメントの随時アップデート

- **内容**: 開発中に決まった仕様変更や、解決したバグの経緯をこまめに `plan.md` へ反映させます。
    
- **メリット**: AIの短期的な記憶容量が限界に達しても、ドキュメントに立ち返ることで一貫性を保てます。

### 8. 車輪の再発明をしない

- **内容**: ユーザー認証やデータベースなど、既存の便利な外部サービス（Supabase等）を積極的に取り入れます。
    
- **メリット**: 複雑な機能を自作する時間を削減し、最も重要なプロダクト独自の価値に集中できます。
